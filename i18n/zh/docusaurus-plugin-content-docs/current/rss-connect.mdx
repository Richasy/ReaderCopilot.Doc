## 支持平台

阅读助理支持以下平台：

- Local（本地管理）
- Google Reader（支持 Google Reader 接口协议的平台，比如 [FreshRSS](https://www.freshrss.org)）
- [Inoreader](https://www.inoreader.com)
- [NewsBlur](https://www.newsblur.com)
- [Feedbin](https://feedbin.com)
- [Miniflux](https://miniflux.app)

## 本地管理

本地管理最为方便，不需要额外授权，应用将会在阅读库中创建 RSS 缓存。

在刷新订阅源的时候，旧的文章不会被删除，而是会一直保留。

可以通过将阅读库放置在如 OneDrive 等本地云盘中以实现跨设备同步。

## Google Reader

Google Reader 已经停运，但是 Google Reader API 作为一种协议却被多种 RSS 平台使用，典型的如 FreshRSS，TheOldReader 等。

以 FreshRSS 举例。

<img src={gReaderAuth} width="340px" />

- **服务器**：指你部署的 FreshRSS 服务的 Google Reader API 地址，通常是：`https://你的FreshRSS部署网址/api/greader.php`
- **用户名**：你在 FreshRSS 中注册的用户名
- **密码**：对应的密码

需要注意的是，如果你是第一次部署，你还需要在 FreshRSS 的设置中的 `认证` 模块，勾选 `允许 API 访问`。

## Inoreader

Inoreader 是一个在线 RSS 服务提供商，算是老牌服务了，质量稳定。

应用继承了 **RSS 追踪** 的客户端令牌，获得了 Inoreader 官方的承认，将继续为 Windows 平台提供高质量的 Inoreader 集成服务。

你可以在设置中切换到 Inoreader，点击 `连接` 按钮，应用会打开浏览器，你需要在浏览器中完成授权，浏览器会引导你再打开阅读助理，这样就完成授权了。

完成授权后，请关闭浏览器中的授权页面。

## NewsBlur

NewsBlur 也是一个老牌 RSS 服务提供商，且提供了完善的 API 接口。

输入你的账户密码即可登录，与 NewsBlur 同步。

但需要说明的是，有些功能 NewsBlur 并没有提供，或者说机制与 阅读助理 现有管理机制不一样。

这部分操作需要到 NewsBlur 中进行（比如删除分组）

## Feedbin

Feedbin 是一个较新的 RSS 服务提供商，API 支持基础的 RSS 阅读及订阅源管理。

通常来说，Feedbin 有自己的官方源，速度快且较为稳定。

但是如果你有自己部署的 Feedbin，也可以将服务器替换为你的 API 地址。

## Miniflux

Miniflux 是一个极简的自部署 RSS 服务。

和 [Google Reader](#google-reader) 类似，你需要填写服务器地址、用户名和密码。

应用使用 `Basic` 授权方式调用 Miniflux 服务的 API，所以会在本地存储你的用户名和密码。

## 不支持的平台

- [Feedly](https://feedly.com/)：热门，质量高，综合能力最强，但对 API 的限制最严格。阅读助理目前不具备申请 API 的资质。
